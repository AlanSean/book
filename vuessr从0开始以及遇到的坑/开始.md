## 开始


### 初始化目录

```
    npm install -g @vue/cli nodemon
        nodemon  检测目录文件更改时，来重启基于node开发的程序
    vue create vuessr
        我选的附带 babel,eslint
    cd vuessr

    下载需要的插件
    npm install --save-dev
        npm-run-all npm运行多个命令 -s 是顺序 -p是并行
        cross-env   可以修改临时环境变量
        webpack-node-externals  解绑所以的node模块
    npm install --save
        vue-server-renderer 不解释
```     

### 修改文件

```
package.json
    eslintConfig
        rules 这个对象下面添加，cli的eslint附带以下的配置 所以手动关闭下。
            "no-console": 0, //关闭  不允许使用 console 理由：测试时，一堆提示烦
            "no-unused-vars": 0, //关闭 未使用的变量 理由：后期会提取插件，框架的源码。所以会触发这种情况
            "no-undef": 0 //关闭 不允许使用未声明的变量 理由：后期会提取插件，框架的源码。所以会触发这种情况

```
